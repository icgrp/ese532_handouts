7.2.2. Review: Burst Transfers and Task-Level Parallelism¶

7.2.2.1. Burst Transfer¶

In HW6, when you partition the matrix multiplication into Load-Compute-Store Pattern (example here), you enabled burst transfer.

The first access to the global memory is expensive while the subsequent contiguous accesses are not. Therefore, in Load block, we copy a chunk of input to the local memory to compute, and similarly in Store block, we copy a chunk of output to global memory. Burst read/write is done with the pipelined loop in Load and Store blocks.

The example of burst read is shown below.

readA:
    for (int i = 0; i < N; i++) {
      for (int j = 0; j < N; j++) {
#pragma HLS PIPELINE
    	  localA[i][j] = A[i * N + j];
      }
    }

7.2.2.2. Task-Level Parallelism¶

This example from previous HW6 also enables task-level parallelism utilizing hls::stream. This adder example is also useful. Key recommendations on hls::stream from Xilinx User Guide are:

  • Data should be transferred in the forward direction only avoiding feedback whenever possible.

  • Each connection should have a single producer and a single consumer.

  • Only the load and store functions should access the primary interface of the kernel
